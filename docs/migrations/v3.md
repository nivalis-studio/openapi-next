# Migrating to v3

## Breaking changes

1. Fluent route builder APIs are removed.
2. Routes now use `defineRoute({ ... })` and export `<yourRoute>.next` (for example, `getUsers.next`).
3. OpenAPI generation now hard-fails on contract issues such as duplicate operation IDs and schema conversion failures.
4. OpenAPI output target is 3.1.

## Before (v2)

```ts
import { getOpenapiOutputs, route } from '@nivalis/openapi-next';
import { z } from 'zod';

const getUsers = route({ method: 'GET', operationId: 'listUsers' })
  .input({ query: z.object({ page: z.coerce.number().int().min(1) }) })
  .outputs(
    getOpenapiOutputs(z.object({
      success: z.literal(true),
      items: z.array(z.object({ id: z.string().uuid() })),
    })),
  );

export const GET = getUsers.action(async ({ query }) =>
  Response.json({ success: true, items: await listUsers(query.page) }),
);
```

## After (v3)

```ts
import { defineRoute } from '@nivalis/openapi-next';
import { z } from 'zod';

const getUsers = defineRoute({
  method: 'GET',
  operationId: 'listUsers',
  input: {
    query: z.object({ page: z.coerce.number().int().min(1) }),
  },
  responses: {
    200: {
      description: 'ok',
      content: {
        'application/json': {
          schema: z.object({
            success: z.literal(true),
            items: z.array(z.object({ id: z.string().uuid() })),
          }),
        },
      },
    },
  },
  handler: async ({ query }) => ({
    status: 200,
    contentType: 'application/json',
    body: { success: true, items: await listUsers(query.page) },
  }),
});

export const GET = getUsers.next;
```
